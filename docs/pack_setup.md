# Pack setup and testing

How the Trashlands modpack is built and how to stand up a test instance. Mirrors the Sky Frogs packwiz workflow.

## What this is

The pack is a [packwiz](https://packwiz.infra.link/) modpack under [`pack/`](../pack). packwiz is git-friendly: each external mod is a small `mods/*.pw.toml` pointer (not the jar), so the repo stays light and reproducible.

- **Target:** Minecraft **26.1.2**, NeoForge **26.1.2.76** (matches the Productive Frogs 2.x line and the Recompile mod - pack and mod must stay on the same versions).
- **Pack metadata:** `pack/pack.toml` (name, version, loader versions), `pack/index.toml` (hash index, regenerated by `packwiz refresh`).

## Layout

```
pack/
  pack.toml          # packwiz manifest (versions, name)
  index.toml         # file hash index (auto-managed - run `packwiz refresh`)
  .packwizignore     # keep tool caches out of the index
  mods/              # one *.pw.toml per external mod; local jars (e.g. Recompile) go here too
  config/            # mod config files, shipped as overrides
tools/
  packwiz-installer-bootstrap.jar   # for building a runnable instance
```

## The core mod: Recompile (local, unpublished)

Recompile is our own mod (sibling repo `../recompile`) and is not on CurseForge/Modrinth, so it ships as a **local jar inside the pack** rather than a `.pw.toml` pointer:

1. Build the jar in the recompile repo (once it has a Gradle project): the jar lands in `../recompile/build/libs/`.
2. Copy `recompile-<version>.jar` into `pack/mods/`.
3. From `pack/`, run `packwiz refresh` - packwiz hashes the jar and adds it to `index.toml`, so it ships with the pack.

Update the jar the same way each build (replace the file, `packwiz refresh`).

## Building a test instance

Two options, same as Sky Frogs:

**A. Prism / MultiMC + packwiz-installer (live-updating):**
1. Create a NeoForge 26.1.2 instance in Prism Launcher.
2. Put `tools/packwiz-installer-bootstrap.jar` in the instance and set it as a pre-launch command pointing at the local `pack/pack.toml` (or a served URL via `packwiz serve`).
3. Launch - the installer syncs mods/config from the pack, including the local Recompile jar.

**B. Export a `.mrpack` (portable):**
1. From `pack/`, run `packwiz modrinth export`.
2. Import the resulting `.mrpack` into Prism Launcher as a new instance.

For iterating on Recompile: rebuild the jar, drop it in `pack/mods/`, `packwiz refresh`, then re-run the installer (option A) or re-launch.

## Version discipline

If Recompile's toolchain lands on a different NeoForge build than `26.1.2.76`, update `pack/pack.toml` `[versions]` to match and `packwiz refresh`. The pack and the mod must never drift apart on loader version.
